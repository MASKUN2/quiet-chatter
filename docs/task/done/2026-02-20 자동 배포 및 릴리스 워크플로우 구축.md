# 2026-02-20 자동 배포 및 릴리스 워크플로우 구축

## 작업 개요
CI/CD 파이프라인을 고도화하여 릴리스 프로세스를 자동화하고, 개발 및 운영 환경의 배포 워크플로우를 분리 및 최적화했습니다.

## 주요 변경 사항

### 1. 릴리스 및 배포 워크플로우 분리
- **`release-please` 도입**: 버전 관리 및 CHANGELOG 생성을 자동화했습니다.
- **워크플로우 분리**:
  - `release-please.yml`: 릴리스 PR 생성 및 태그 발행 담당.
  - `deploy-dev.yml`: `dev` 브랜치 푸시 시 개발 서버 자동 배포.
  - `deploy-prod.yml`: 릴리스 태그(`v*`) 발행 시 운영 서버 자동 배포.

### 2. 보안 및 설정 개선
- **민감 정보 보호**: `application.yml` 등의 설정 파일에서 민감한 정보를 환경 변수로 대체하거나 암호화했습니다.
- **배포 스크립트 최적화**: Docker Compose 기반의 무중단 배포 스크립트(`deploy.sh`)를 개선하여 환경별 설정을 유연하게 적용하도록 했습니다.

### 3. 문서화
- **인프라 가이드 업데이트**: 프론트엔드 DNS 정보 추가 및 인프라 다이어그램(Mermaid) 문법 오류 수정.
- **에이전트 가이드 개선**: 작업 절차 및 워크플로우 관련 가이드를 명확하게 업데이트했습니다.

## 검증
- Github Actions 워크플로우 실행 테스트 완료.
- 개발 및 운영 환경 배포 시나리오 검증 완료.

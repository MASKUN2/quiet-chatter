1. 기능에 대한 설명 문서를 markdown으로 /docs/feature/ 경로에 파일명을 한국어로 저장한다.
2. 문서의 목차는 "요구사항", "구현개요", "다이어그램", "상세구현내용" 순으로 작성한다.
3. 요구사항을 파악하여 "요구사항"에 정리한다.
4. 코드베이스에서 해당 기능을 구현한 코드를 분석을 하고 "구현개요"를 작성한다.
5. 해당 기능의 프로세스를 도식화하여 mermaid 문법으로 "다이어그램"에 작성한다.
6. 해당 기능의 상세한 구현 내용을 "상세구현내용"에 작성한다.
7. 이 문서가 요구사항 명세서의 특정 기능 이름을 바탕으로 작성될 경우 해당 기능이름에 문서를 링크한다.

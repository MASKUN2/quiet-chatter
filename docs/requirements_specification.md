# 요구사항 명세서 (Requirements Specification)

## 1. 기능 요구사항 (Functional Requirements)

### 도서 및 저자 관리

- 사용자는 키워드(제목, 저자 등)로 책을 검색할 수 있다.
- 사용자는 책의 저자를 검색하고, 해당 저자의 도서 목록을 확인할 수 있다.
- 외부 도서 API(Naver 등)를 연동하여 도서 정보를 통합 관리한다.

### 북톡 (Talk) 및 커뮤니티

- 각 사용자는 책 또는 저자에 대해 댓글 형식의 '톡'을 작성, 수정, 삭제할 수 있다.
- '톡'은 250자 이내로 작성 가능하다.
- 사용자는 다른 사용자가 작성한 '톡'에 반응(좋아요, 공감해요)할 수 있다.
- 사용자는 다른 사람들이 작성한 '톡'을 추천받을 수 있다.
- 목록 조회 시 무한 스크롤을 지원한다.

### 정책 및 접근 권한

- 서비스의 모든 기능은 비로그인 사용자의 접근을 허용한다.
- 기본적으로 작성 후 1년이 지난 '톡'은 자동으로 비공개(숨김) 처리된다.

## 2. 도메인 모델 (Domain Model)

- **책 (Book)**: ISBN, 제목, 출판사, 저자 정보를 포함한다.
- **멤버 (Member)**: 게스트, 정회원, 어드민으로 구분된다.
- **북톡 (Talk)**: 250자 이내의 본문, 자동 숨김 정책을 위한 생성 시간을 가진다.
- **리액션 (Reaction)**: 북톡에 대한 사용자 반응 (좋아요, 공감해요).
- **자동 숨김 (Self Hidden)**: 생성 후 1년 경과 시 활성화되는 상태 값.
